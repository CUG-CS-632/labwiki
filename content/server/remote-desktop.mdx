# 远程桌面

两台服务器都支持图形化远程桌面连接：Windows GPU 服务器使用 Windows 原生远程桌面，Linux CPU 服务器通过 XRdp 提供远程桌面服务。

## 连接信息

| 服务器 | 地址 | 桌面服务 | 适用场景 |
|--------|------|----------|----------|
| Windows GPU 服务器 | `lab632g.pibonds.com` | Windows 原生远程桌面 | Windows 图形界面、GPU 任务管理 |
| Linux CPU 服务器 | `lab632.pibonds.com` | XRdp | Linux 图形界面、MATLAB 等 GUI 工具 |

优先使用域名连接。服务器 IP 可能变化，Wiki 不记录固定 IP；确需 IP 时请登录对应服务器查询当前地址。

## 连接前准备

- 已获得服务器账号和密码。
- 当前网络可以访问实验室内网；校外访问通常需要先连接 VPN。
- 本地电脑已安装或启用远程桌面客户端。

## Windows 客户端

Windows 系统自带远程桌面连接工具，无需安装额外软件。

### 连接步骤

| 目标服务器 | 输入内容 |
|------------|----------|
| Windows GPU 服务器 | `lab632g.pibonds.com` |
| Linux CPU 服务器 | `lab632.pibonds.com` |

1. 按 `Win + R`，输入 `mstsc`，回车打开远程桌面连接
2. 在「计算机」栏输入目标服务器域名
3. 点击「连接」
4. 输入服务器的用户名和密码
5. 连接 Linux CPU 服务器时，进入 XRdp 会话后即可使用 Linux 图形界面

### 保存连接配置

为了避免每次输入地址，可以保存为 `.rdp` 文件：

1. 输入计算机地址后，点击「显示选项」
2. 输入用户名
3. 点击「另存为」保存到桌面
4. 双击 `.rdp` 文件即可快速连接

## macOS 客户端

macOS 需要安装微软发布的 **Windows** 应用，用它进行远程桌面连接。

### 安装应用

1. 打开 App Store
2. 搜索 **Windows**
3. 安装并打开应用

### 连接步骤

1. 打开 **Windows** 应用
2. 点击右上角 **+** 按钮，选择「Add PC」
3. 在「PC name」栏输入目标服务器域名：`lab632g.pibonds.com` 或 `lab632.pibonds.com`
4. 在「User account」栏选择「Add User Account」，输入用户名和密码
5. 点击「Add」保存配置
6. 双击刚添加的 PC 即可连接

### 常用设置

- **显示分辨率**：在 PC 设置中可调整显示分辨率
- **共享文件夹**：可在本地资源中设置共享文件夹，方便文件传输
- **全屏模式**：连接后可通过顶部工具栏切换全屏

## 使用建议

- 运行长时间程序时，关闭窗口前选择断开连接即可保留会话；不要随意注销其他人的会话。
- 大文件传输优先使用 [SMB 文件共享](/server/smb)，不要依赖剪贴板复制大量文件。
- 如果桌面卡顿，先降低分辨率或关闭多显示器映射，再重新连接。

## 常见问题

### 无法连接

- 确认当前网络能访问实验室内网。
- 先尝试 `ping lab632g.pibonds.com` 或 `ping lab632.pibonds.com` 检查域名是否可达。
- 确认本地防火墙或 VPN 没有阻断 RDP。

### 提示账号或密码错误

- 检查用户名是否包含域或主机名前缀；如果管理员给出了固定格式，按管理员提供的格式填写。
- 确认键盘布局没有导致特殊字符输入错误。

### 证书警告

首次连接时客户端可能提示证书无法验证。如果服务器地址正确，可以继续连接；不要对未知地址忽略证书警告。
