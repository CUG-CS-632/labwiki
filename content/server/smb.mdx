# SMB 文件共享

Windows GPU 服务器 `lab632g.pibonds.com` 提供 SMB 文件共享服务。当前共享名为 `LabSMB`，可用于访问个人私有目录和实验室公共软件目录。

## 连接信息

| 项目 | 信息 |
|------|------|
| 服务器 | Windows GPU 服务器 |
| 域名 | `lab632g.pibonds.com` |
| 共享名 | `LabSMB` |
| 协议 | SMB |
| 默认端口 | `445` |
| Windows 访问地址 | `\\lab632g.pibonds.com\LabSMB` |
| macOS 访问地址 | `smb://lab632g.pibonds.com/LabSMB` |

服务器 IP 可能变化，建议优先使用域名连接。确需 IP 时，请先在 Windows GPU 服务器上查询当前地址。

## 目录说明

| 目录 | 说明 |
|------|------|
| `<你的用户名>` | 个人私有目录。只能看到并访问自己的用户名目录，无法看到其他用户的私有目录。 |
| `Public` | 公共目录，放置 Windows 科研常用软件。需要时可从这里下载。 |

## Windows 客户端

### 直接访问

1. 打开文件资源管理器。
2. 在地址栏输入：

```
\\lab632g.pibonds.com\LabSMB
```

3. 按回车访问共享目录。
4. 按提示输入服务器账号和密码。
5. 进入后可访问自己的私有目录和 `Public` 目录。

个人文件请放在以自己用户名命名的目录中；`Public` 主要用于下载公共软件。

### 映射网络驱动器

如果经常访问共享目录，可以把它映射为本地盘符：

1. 打开文件资源管理器。
2. 右键点击「此电脑」，选择「映射网络驱动器」。
3. 选择驱动器盘符，例如 `Z:`。
4. 在文件夹栏输入：

```
\\lab632g.pibonds.com\LabSMB
```

5. 可按需勾选「登录时重新连接」。
6. 点击「完成」，并按提示输入服务器账号和密码。

### 命令行映射

也可以在 PowerShell 或 CMD 中使用命令映射：

```powershell
net use Z: \\lab632g.pibonds.com\LabSMB /user:username
```

其中 `username` 替换为你的服务器账号。断开映射可使用：

```powershell
net use Z: /delete
```

## macOS 客户端

### Finder 连接

1. 打开 Finder
2. 按 `Cmd + K` 打开「连接服务器」窗口
3. 在服务器地址栏输入：`smb://lab632g.pibonds.com/LabSMB`
4. 点击「连接」
5. 选择「注册用户」，输入用户名和密码
6. 进入后可访问自己的私有目录和 `Public` 目录

### 在 Finder 中查看

连接成功后，共享文件夹会出现在：

- Finder 侧边栏的「网络」下方
- 桌面上（如果设置了显示网络卷）

### 保存收藏

为了快速访问，可以将共享地址添加到收藏：

1. 连接服务器后，在 Finder 侧边栏右键点击共享文件夹
2. 选择「添加到边栏」
3. 或者在「连接服务器」窗口点击 **+** 按钮保存地址

### 命令行挂载

在终端中也可以手动挂载共享目录：

```bash
mkdir -p /Volumes/LabSMB
mount_smbfs //username@lab632g.pibonds.com/LabSMB /Volumes/LabSMB
```

其中 `username` 替换为你的服务器账号。卸载可使用：

```bash
umount /Volumes/LabSMB
```

## 传输文件

连接共享后，可以直接在文件管理器中拖拽文件进行传输。个人文件建议放入 `<你的用户名>` 目录，不要直接堆放在共享根目录或 `Public` 目录。

也可以使用命令行复制文件：

```bash
# macOS
cp localfile.txt /Volumes/LabSMB/username/
```

```powershell
# Windows PowerShell
Copy-Item localfile.txt Z:\username\
```

大量文件同步时可以使用 `rsync`：

```bash
rsync -av /local/path/ /Volumes/LabSMB/username/remote/path/
```

上面示例中的 `username` 替换为你的服务器账号。

## 使用建议

- 大文件传输优先使用有线网络或稳定的校园网。
- 传输大量小文件前可以先打包压缩，减少 SMB 往返开销。
- 不要把共享目录当作唯一备份；重要数据需要保留额外副本。
- 个人目录用于存放自己的文件，`Public` 目录用于下载公共软件，不建议在 `Public` 中放置个人数据。

## 常见问题

### 无法连接

- 确认和服务器在同一局域网内
- 检查服务器的 SMB 服务是否正常运行
- Windows 防火墙可能需要放行 SMB 端口（445）
- 如果需要用 IP 排查，请先在 Windows GPU 服务器上查询当前 IP 地址

### 传输速度慢

- 检查网络连接质量
- 大文件传输建议使用有线网络
- 可以尝试压缩后再传输

### 权限问题

- 确认使用的是服务器账号和密码。
- 个人私有目录按用户名隔离，看不到其他用户目录是正常现象。
- 如果无法看到自己的目录或无法访问 `Public`，联系管理员确认权限设置。
