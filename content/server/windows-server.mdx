# Windows GPU 服务器

`LAB632G` 是实验室的 Windows GPU 服务器，适合 GPU 计算、Windows 图形化软件和 SMB 文件共享。需要桌面环境时使用 Windows 原生远程桌面，需要命令行或 VSCode 远程连接时使用 SSH。

## 基本信息

| 项目 | 信息 |
|------|------|
| 主机名 | `LAB632G` |
| 域名 | `lab632g.pibonds.com` |
| 操作系统 | Windows Server 2025 Standard 24H2 x86_64 |
| 设备型号 | Dell Precision 7920 Tower |
| 区域设置 | `zh-CN` |

## 查询当前 IP

服务器 IP 可能变化，Wiki 不记录固定 IP。需要确认当前地址时，可登录 Windows GPU 服务器后在 CMD 或 PowerShell 中运行：

```powershell
ipconfig
```

## 硬件配置

| 组件 | 规格 |
|------|------|
| CPU | 2 × Intel Xeon Gold 6246R，最高约 4.10 GHz |
| GPU | 2 × NVIDIA GeForce RTX 3090，每张约 24 GiB 显存 |
| 内存 | 约 384 GiB |
| 系统盘 | `C:\`，约 1.75 TiB，NTFS |
| 数据盘 | `D:\`、`E:\`、`F:\`，每个约 10.91 TiB，NTFS |

## 存储建议

| 位置 | 建议用途 |
|------|----------|
| `C:\` | 系统、软件、少量用户配置，不建议长期堆放数据 |
| `D:\LabUsers\${username}` | 个人数据、项目文件、实验结果，推荐优先使用 |
| `D:\` / `E:\` / `F:\` | 大型数据集、共享数据等按管理员规划使用的目录 |

通过 SMB 访问共享目录的方法见 [SMB 文件共享](/server/smb)。

## 适用场景

- 需要 GPU 加速的深度学习训练和推理
- 既需要 GPU 又需要 Linux 环境时，可在本机自行安装 WSL 使用
- 大文件存储和共享
- Windows 专属软件的运行
- 需要可视化桌面的实验、标注或管理任务

## 使用注意

- 运行长时间 GPU 任务前，先确认两张 RTX 3090 的占用情况，避免影响他人任务。
- 个人数据建议存放在 `D:\LabUsers\${username}`，不要堆在系统盘。
- 不要使用管理员权限安装任何软件，避免引入病毒或木马。
- 如果确认必须使用管理员权限才能完成工作，请先联系管理员。
- 通过 RDP 断开连接时，优先选择“断开”而不是直接注销，避免意外结束正在运行的桌面任务。
- 重要实验数据需要自行确认备份策略；共享盘不应默认视为长期备份。
