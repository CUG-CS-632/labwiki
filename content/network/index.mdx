# 校园网

本文面向个人电脑等用户客户端，处理校园网认证、IP 获取、DNS 解析和自动登录问题。这里的内容只针对客户端联网，和实验室服务器使用说明无关。

## 认证入口

校园网认证页面是：

```text
nap.cug.edu.cn
```

连接校园网 Wi-Fi 或插入校园网网线后，浏览器通常会自动跳转到认证页面。如果没有跳转，手动访问 `nap.cug.edu.cn`。

## 推荐处理顺序

无法认证时，按下面顺序排查，避免一开始就修改很多网络设置。

### 1. 关闭代理

校园网认证流量需要直接访问学校网关。无法打开认证页、认证后仍不能上网、CampusLink 报连接失败时，先关闭：

- 系统代理
- 浏览器代理或代理扩展
- VPN
- TUN 模式、透明代理、虚拟网卡代理
- 代理工具接管 DNS 的功能

关闭后重新连接校园网，再访问 `nap.cug.edu.cn` 或重新运行 CampusLink。

### 2. 检查是否拿到 IP

确认代理没问题后，先看本机是否已经从校园网 DHCP 获取到 IP。

| 系统 | 查看命令 |
|------|----------|
| Windows | `ipconfig` |
| macOS | `ifconfig` |
| Linux | `ip addr` |

如果没有校园网分配的 IPv4 地址，重新向 DHCP 申请地址。

Windows PowerShell 或 CMD：

```powershell
ipconfig /release
ipconfig /renew
```

macOS：

```bash
sudo ifconfig en0 down
sudo ifconfig en0 up
```

多数 Mac 的 Wi-Fi 是 `en0`。如果不确定网卡名，先运行 `networksetup -listallhardwareports` 查看。

Linux NetworkManager：

```bash
nmcli networking off
nmcli networking on
```

桌面环境中也可以直接断开并重新连接当前校园网。

### 3. 检查 DNS 和网关地址

如果 IP 正常，但打不开 `nap.cug.edu.cn`，优先检查 DNS。校园网认证页面可能依赖 DHCP 下发的默认 DNS；手动设置公共 DNS、代理 DNS 或固定 DNS 时，可能无法访问认证页。

推荐先把 DNS 改回自动获取，然后断开并重新连接校园网。

常用查看方式：

| 系统 | 查看方式 |
|------|----------|
| Windows | `ipconfig /all` |
| macOS | 系统设置 → 网络 → 当前网络 → 详细信息 → DNS |
| Linux | `resolvectl status` |

如果暂时不能改 DNS，可以临时设置 hosts。认证网关 IP 通常是 `192.168.167.115` 和 `192.168.167.116`，任选一个可达地址即可：

```text
192.168.167.115 nap.cug.edu.cn
```

hosts 文件位置：

| 系统 | hosts 路径 |
|------|------------|
| Windows | `C:\Windows\System32\drivers\etc\hosts` |
| macOS | `/etc/hosts` |
| Linux | `/etc/hosts` |

hosts 只适合临时排障。长期使用建议恢复 DHCP 默认 DNS，或使用 CampusLink 的备用网关参数。

## CampusLink 自动登录

[cugcs632/CampusLink](https://github.com/cugcs632/CampusLink) 是校园网 SRun 自动登录工具，当前默认面向 `nap.cug.edu.cn`。它是 Go 编写的单文件二进制程序，不需要 Python、Node.js、包管理器或虚拟环境。

适合这些场景：

- 每次开机、睡眠恢复或 Wi-Fi 重连后不想手动打开认证页
- 认证页受 DNS 或浏览器代理影响，经常打不开
- 希望定时重试，覆盖网关掉线或切换网络后的重新认证

### 下载

从 [CampusLink Releases](https://github.com/cugcs632/CampusLink/releases/latest) 下载对应系统的压缩包。

| 系统 | 下载包 | 程序文件 |
|------|--------|----------|
| Windows x64 | `campuslink-windows-amd64.zip` | `campuslink.exe` |
| macOS Apple Silicon | `campuslink-darwin-arm64.tar.gz` | `campuslink` |
| Linux x64 | `campuslink-linux-amd64.tar.gz` | `campuslink` |

推荐安装位置：

| 系统 | 路径 |
|------|------|
| Windows | `%LOCALAPPDATA%\CampusLink\campuslink.exe` |
| macOS | `$HOME/.local/bin/campuslink` |
| Linux | `$HOME/.local/bin/campuslink` |

### 首次测试

先手动验证账号、密码和认证参数。

Windows PowerShell：

```powershell
& "$env:LOCALAPPDATA\CampusLink\campuslink.exe" -u "学号" -p "密码"
```

macOS/Linux：

```bash
"$HOME/.local/bin/campuslink" -u "学号" -p "密码"
```

成功时会显示 `login ok`。如果要查看网关返回的原始信息，加 `--json`：

```bash
"$HOME/.local/bin/campuslink" -u "学号" -p "密码" --json
```

### 保存账号

为了避免把密码留在命令历史里，可以用环境变量保存账号。

Windows PowerShell：

```powershell
[Environment]::SetEnvironmentVariable("SRUN_USERNAME", "学号", "User")
[Environment]::SetEnvironmentVariable("SRUN_PASSWORD", "密码", "User")
```

关闭并重新打开 PowerShell 后测试：

```powershell
& "$env:LOCALAPPDATA\CampusLink\campuslink.exe"
```

macOS/Linux：

```bash
export SRUN_USERNAME="学号"
export SRUN_PASSWORD="密码"
"$HOME/.local/bin/campuslink"
```

长期自动运行时，请按 CampusLink README 使用 Windows 任务计划程序、Linux systemd 用户 timer 或 macOS LaunchAgent。推荐策略是登录或开机后执行一次，并每 5 分钟重试一次；CampusLink 会把“已经在线”视为成功，因此重复执行是安全的。

### 常用参数

| 环境变量 | 命令行参数 | 用途 |
|----------|------------|------|
| `SRUN_USERNAME` | `-u`, `--username` | 校园网账号 |
| `SRUN_PASSWORD` | `-p`, `--password` | 校园网密码 |
| `SRUN_IP` | `--ip` | 自动识别客户端 IP 失败时手动指定 |
| `SRUN_HOST` | `--host` | 指定认证网关，默认 `nap.cug.edu.cn` |
| `SRUN_AC_ID` | `--ac-id` | 接入点 ID，默认 `1` |
| `SRUN_TIMEOUT` | `--timeout` | HTTP 超时时间，单位为秒 |

如果 DNS 问题导致默认网关域名不可访问，可以临时指定备用网关：

```bash
"$HOME/.local/bin/campuslink" --host 192.168.167.115
"$HOME/.local/bin/campuslink" --host 192.168.167.116
```

Windows PowerShell：

```powershell
& "$env:LOCALAPPDATA\CampusLink\campuslink.exe" --host 192.168.167.115
& "$env:LOCALAPPDATA\CampusLink\campuslink.exe" --host 192.168.167.116
```

## CampusLink 常见报错

| 报错 | 优先处理 |
|------|----------|
| `cannot find client ip from portal page; pass --ip` | 重新从 DHCP 获取 IP；仍失败时用 `--ip` 或 `SRUN_IP` 手动指定客户端 IP |
| `portal connection failed` | 检查是否连接校园网、代理是否关闭、DNS 是否能解析 `nap.cug.edu.cn`；必要时使用 `--host` 指定备用网关 |
| `login failed` | 加 `--json` 查看网关原始错误，再确认账号、密码、`ac_id` 和网关地址 |

## 凭据安全

CampusLink 的环境变量、Linux `EnvironmentFile`、macOS LaunchAgent plist 和 Windows 用户级环境变量都会在本机保存密码。请确保本机系统账号有登录密码，不要把包含校园网账号密码的配置文件提交到 Git 或发给他人。
